## 1. 概述

`cost_aware_straighten_lcp.py` 提供了一种最低成本路径算法，能在栅格成本表面上
生成视觉整洁、成本高效的路径。它作为 **方案C（Approach C）** 开发，旨在消除基于
栅格寻路固有的"闪电状"折线伪影，同时确保拉直后的路径不会悄悄穿越高成本区域。

核心思想是：

> 保留稳健的八方向 Dijkstra 搜索，但**升级后处理拉直步骤**——对每一段候选捷径
> 不仅检查是否穿越 NODATA 障碍，还将捷径的累积代价与原始路径代价进行比较，只有
> 当比值在容差范围内时才接受捷径。

---

## 2. 为什么需要代价感知拉直？

标准的八连通栅格寻路产生的路径只能沿 8 个方向（N、NE、E、SE、S、SW、W、NW）移
动，这会产生阶梯状/锯齿状折线伪影，看起来不自然。

原始版本 `enhanced_lcp.py` 通过**视线拉直（line-of-sight straightening）**后处
理来解决此问题：尝试跳过中间航点并绘制直线，只拒绝穿越 NODATA / 障碍像元的捷径。

然而，"仅检查 NODATA"存在盲区：捷径线可能穿过*有效*但代价非常高的像元。结果是
一条视觉上笔直但经济性很差的路径。

**代价感知拉直**弥补了这一缺陷。在接受捷径之前，它会计算捷径线上的累积代价，并
将其与同一段的原始栅格路径代价进行比较。只有在可配置的容差比率内的捷径才会被接受。

```
原始栅格路径：    A ──► B ──► C ──► D ──► E
候选捷径：        A ─────────────────────► E

接受捷径的条件：
  cost(捷径) ≤ cost(A→B→C→D→E) × cost_tolerance
```

## 3. 算法流水线

算法依次执行三个阶段。

### 3.1 第一步——Dijkstra 路径搜索

使用 **八方向改进型 Dijkstra / A\*** 搜索，在成本栅格上寻找从起点到终点的最优
栅格路径。此步骤与 `enhanced_lcp.py` 完全相同，支持：

- **基础代价**：`cost_raster[像元] × 步长距离`
- **弯曲度惩罚**（当 `curvature_factor > 0` 时生效）
- **硬转弯约束**（当 `max_turning_angle < 180` 时生效）
- **防锯齿直线惩罚**：当相邻像元代价相近时，抑制方向变化
- **距离惩罚**（当 `distance_factor > 0` 时生效）

当不需要弯曲度控制（`curvature_factor == 0` 且 `max_turning_angle == 180`）
时，算法自动切换到更简单的无方向 Dijkstra 变体，降低内存消耗。

### 3.2 第二步——代价感知拉直

栅格路径找到后，**代价感知拉直**移除不必要的航点：

1. **累积代价预计算**：沿栅格路径行走，逐步累加
   `cost_raster[像元] × 步长距离`。这使得任意子段的原始路径代价可以在 O(1) 内
   查询。

2. **贪心捷径扫描**：从第一个航点开始，向前搜索最远
   `max_skip = max(2, int(straighten_factor × (n − 1)))` 个位置（从最远处
   开始扫描）。对每个候选终点依次检查：
   - **障碍检查**：使用超覆盖线（supercover line）算法验证直线是否穿越任何
     NODATA / 越界像元。
   - **代价检查**：计算捷径线上的积分代价（平均像元代价 × 欧几里得距离），与
     `原始段代价 × cost_tolerance` 进行比较。
   - 两项检查均通过时，接受捷径并跳转到新位置。

3. 结果是一组精简的航点，由直线段连接，既可通行又在经济上可接受。

### 3.3 第三步——Chaikin 平滑

对拉直后的路径施加 **Chaikin 角切割算法**（默认 3 次迭代），将尖锐的转角替换为
平滑的弧线。平滑后会进行 NODATA 安全检查，验证每一段是否穿越了障碍像元；如果任
何平滑后的线段穿越了障碍，则回退到未平滑的拉直路径。

## 4. 代价函数详解

Dijkstra 搜索中每一步的代价为：

```
step_cost = base_cost
          + curvature_penalty
          + straightness_penalty
          + distance_penalty
```

| 组成部分 | 公式 | 生效条件 |
|---|---|---|
| **base_cost（基础代价）** | `cost_raster[B] × step_distance` | 始终生效 |
| **curvature_penalty（弯曲度惩罚）** | `curvature_factor × 5.0 × cost_scale × (angle / 180) × step_distance` | `curvature_factor > 0` |
| **straightness_penalty（直线惩罚）** | `similarity × 0.3 × cost_scale × step_distance` | 方向发生变化且使用方向感知模式 |
| **distance_penalty（距离惩罚）** | `distance_factor × cost_scale × step_distance` | `distance_factor > 0` |

- **step_distance**——两个像元中心之间的欧几里得距离（正方向为 1.0，对角线为 √2，
  经 `cell_size` 缩放）。
- **cost_scale**——所有有限非负栅格值的均值。用于将惩罚项归一化到与基础代价相同
  的数量级。
- **similarity**——`max(0, 1 − |cost_target − cost_straight| / cost_scale)`。
  当目标像元与直行像元的代价相近时值高；当代价差异大时抑制防锯齿惩罚。

## 5. 参数详解与调参建议

### 5.1 `cost_raster`（成本栅格）

| | |
|---|---|
| **类型** | 二维 NumPy 数组（`float32` 或 `float64`） |
| **必填** | 是 |
| **说明** | 遍历成本表面。每个像元的值代表穿越该像元的代价。`NaN` 和 `Inf` 被视为不可通行的障碍。负值也被视为障碍。 |

**调参注意事项**：
- 确保障碍像元使用 `NaN`，而不是零（零是合法的低代价值）。
- 支持大栅格（> 5000 × 5000），但可能需要数秒计算时间。使用 `float32` 可减少
  内存占用。

---

### 5.2 `start` / `end`（起点 / 终点）

| | |
|---|---|
| **类型** | `(row, col)` 整数元组 |
| **必填** | 是 |
| **说明** | 路径的起点和终点，使用从零开始的行/列索引。两者必须在栅格范围内，且位于有限代价的像元上。 |

---

### 5.3 `curvature_factor`（弯曲度因子）

| | |
|---|---|
| **类型** | `float` |
| **范围** | 0.0 – 1.0 |
| **默认值** | 0.0 |
| **说明** | 对急转弯施加的软惩罚权重。值越高，路径曲线越平滑，但总成本可能增加。 |

**调参建议**：

| 值 | 效果 |
|---|---|
| **0.0** | 无弯曲度惩罚——标准 LCP 行为。速度最快。 |
| **0.1 – 0.3** | 轻度平滑。减少最极端的锯齿，不会显著改变路线。大多数场景的推荐起点。 |
| **0.4 – 0.6** | 中度平滑。路径明显避免急转弯，但可能绕过成本谷地。 |
| **0.7 – 1.0** | 激进平滑。曲线非常平缓，适用于管道、高速公路等需要大转弯半径的场景。预期成本会明显增加。 |

> **提示**：从 `0.0` 开始，每次增加 `0.1`，直到视觉效果满足需求。可与
> `max_turning_angle` 配合使用来设置硬约束。

---

### 5.4 `max_turning_angle`（最大转弯角度）

| | |
|---|---|
| **类型** | `float` |
| **范围** | 0 – 180 度 |
| **默认值** | 180.0 |
| **说明** | 每个栅格步骤允许的最大转弯角度硬限制。超过此角度的转弯在搜索过程中完全禁止。 |

**调参建议**：

| 值 | 含义 |
|---|---|
| **180** | 无约束——所有转弯均允许。 |
| **135** | 禁止掉头（180° 转弯）。轻度约束。 |
| **90** | 只允许前进或侧向移动，不允许后退。产生大弯度路径。 |
| **45** | 只允许直行或微转。非常严格——在复杂栅格上可能找不到路径。 |

> **警告**：过低的值（< 45°）可能导致"找不到路径"错误。建议同时配合
> `curvature_factor` 使用，而不是单独依赖此参数。

---

### 5.5 `distance_factor`（距离因子）

| | |
|---|---|
| **类型** | `float` |
| **范围** | 0.0 – 1.0 |
| **默认值** | 0.0 |
| **说明** | 欧几里得路径长度在代价函数中的权重。值越高，算法越倾向于选择更短的路径，即使需要穿越略贵的像元。 |

**调参建议**：

| 值 | 效果 |
|---|---|
| **0.0** | 纯代价优化——无论路径多长，只要代价最低。 |
| **0.1 – 0.3** | 轻度缩短。消除主要绕行，同时仍优先选择低成本走廊。推荐起始范围。 |
| **0.4 – 0.6** | 平衡：代价和距离大致等权。 |
| **0.7 – 1.0** | 强烈距离偏好。路径趋向直线；代价权重被削弱。 |

> **提示**：如果路径在低成本区域绕了一大圈，尝试 `distance_factor = 0.2` 将其
> 拉回更直接的路线。

---

### 5.6 `straighten_factor`（拉直因子）

| | |
|---|---|
| **类型** | `float` |
| **范围** | 0.0 – 0.5 |
| **默认值** | 0.3 |
| **说明** | 控制拉直步骤向前搜索的距离。内部转换为最大前瞻量 `max(2, int(factor × (n − 1)))`，其中 `n` 是路径总节点数。 |

**调参建议**：

| 值 | 效果 |
|---|---|
| **0.0** | 不拉直——输出保留所有八方向栅格步骤。可用于检查原始 Dijkstra 路径。 |
| **0.05 – 0.15** | 保守：只拉直非常短的锯齿段。路径与原始栅格路径非常接近。 |
| **0.2 – 0.3** | 适中（默认范围）：产生视觉整洁的路径，同时尊重成本障碍。大多数场景推荐。 |
| **0.4 – 0.5** | 激进：允许长距离直线捷径。适合开阔、均匀成本区域，但可能过度简化复杂地形中的路径。 |

> **与 `cost_tolerance` 的联动**：大的 `straighten_factor` 只有在
> `cost_tolerance` 也允许较长捷径时才有效。如果 `cost_tolerance` 非常严格
> （如 1.0），即使前瞻很远的捷径也会因为代价略高而被拒绝。

---

### 5.7 `cost_tolerance`（代价容差）

| | |
|---|---|
| **类型** | `float` |
| **范围** | ≥ 1.0 |
| **默认值** | 1.05 |
| **说明** | 捷径代价与原始栅格路径代价之比的最大允许值。**这是本工具的核心独有参数。** |

**工作原理**：

```
接受捷径 A → E  ⟺  cost(捷径) ≤ cost(A→…→E) × cost_tolerance
```

**调参建议**：

| 值 | 含义 | 适用场景 |
|---|---|---|
| **1.0** | 只接受代价与原始路径相同或更低的捷径。非常严格——在非均匀表面上可能几乎不拉直。 | 绝对成本最优至关重要时。 |
| **1.01 – 1.05** | 允许 1 – 5% 的代价增加。视觉质量与成本效率之间的良好平衡。**推荐默认范围。** | 通用场景。 |
| **1.05 – 1.15** | 允许 5 – 15% 的代价增加。更激进的拉直；路径视觉更整洁，但可能付出适度的成本溢价。 | 视觉直线度比最小成本更重要时。 |
| **1.2 – 1.5** | 允许 20 – 50% 的代价增加。强烈偏好直线路径；可能有显著成本增加。 | 偏重美观或成本表面变化较小时。 |
| **2.0+** | 基本上禁用代价检查——行为与 `enhanced_lcp.py` 中仅检查 NODATA 的拉直相同。 | 想要最大程度拉直，不在乎代价时。 |

> **提示**：大多数 GIS 应用建议从默认值 `1.05` 开始。如果路径仍有明显锯齿，增加
> 到 `1.10`。如果路径不可接受地穿越了高成本区域，降低到 `1.02` 或 `1.0`。

---

### 5.8 `cell_size`（像元尺寸）

| | |
|---|---|
| **类型** | `(y_size, x_size)` 浮点元组 |
| **默认值** | `(1.0, 1.0)` |
| **说明** | 每个栅格像元在地图单位下的物理尺寸（如米）。用于计算步长代价、距离惩罚和拉直线积分代价的准确欧几里得距离。 |

**调参注意事项**：
- 如果使用 ArcGIS 工具箱包装器，`cell_size` 会从输入栅格中自动读取。
- 独立使用时，通过栅格元数据（如 `rasterio`）获取像元尺寸。
- 确保 `y_size` 和 `x_size` 的单位与成本栅格值的预期缩放一致。

## 6. 输出结果字典

函数返回一个字典，包含以下键：

| 键 | 类型 | 说明 |
|---|---|---|
| `path` | `list[(int, int)]` | 从起点到终点的栅格路径（八连通）。 |
| `straightened_path` | `list[(float, float)]` | 代价感知拉直后的路径——精简航点，以直线段连接。 |
| `smoothed_path` | `list[(float, float)]` | 拉直路径经 Chaikin 平滑后的版本，转角被圆弧替代。 |
| `total_cost` | `float` | 沿最优栅格路径的累积代价。 |
| `path_length` | `float` | 栅格路径的物理长度（地图单位）。 |
| `directions` | `list[int]` | 栅格路径每一步的方向索引（0–7）。 |
| `turning_angles` | `list[float]` | 栅格路径每个内部顶点的转弯角度（度）。 |

## 7. 快速入门示例

```python
import numpy as np
from cost_aware_straighten_lcp import cost_aware_least_cost_path

# 创建示例成本栅格
raster = np.random.default_rng(42).uniform(1, 10, (100, 100))

result = cost_aware_least_cost_path(
    raster,
    start=(0, 0),
    end=(99, 99),
    curvature_factor=0.2,
    max_turning_angle=90.0,
    distance_factor=0.1,
    straighten_factor=0.3,
    cost_tolerance=1.05,
)

print(f"栅格路径节点数    : {len(result['path'])}")
print(f"拉直后航点数      : {len(result['straightened_path'])}")
print(f"平滑后点数        : {len(result['smoothed_path'])}")
print(f"总代价            : {result['total_cost']:.2f}")
print(f"路径长度          : {result['path_length']:.2f}")
```

## 8. 与 enhanced_lcp.py 的对比

| 特性 | `enhanced_lcp.py` | `cost_aware_straighten_lcp.py` |
|---|---|---|
| Dijkstra 搜索 | 八方向、弯曲度感知 | **完全相同** |
| 拉直检查内容 | 仅检查 NODATA 障碍 | NODATA 障碍 **+ 累积代价比较** |
| 额外参数 | — | `cost_tolerance`（≥ 1.0） |
| 拉直后穿越高成本区域的风险 | 可能发生 | 受 `cost_tolerance` 控制 |
| 性能 | 略快（拉直阶段无代价积分计算） | 略慢（每个捷径候选需进行超覆盖线代价求和） |
| 最佳适用场景 | 均匀或低变化的成本表面 | 非均匀表面，需要成本保真度时 |
